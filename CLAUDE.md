## 1. プロジェクト概要

- **プロジェクト名**：加賀LOMアカデミー事業 議案構築プロジェクト
- **発議元**：公益社団法人 加賀青年会議所（2026年度）会員開発委員会
- **対象議案**：会員開発委員会アカデミー事業（既存会員のJC活動参加率向上）
- **議案上程予定**：2026年5月24日（理事会）／確認日：2026年5月26日
- **担当**：会員開発委員会 室長 中谷一世 ／ 委員長 藤澤祐希

## 2. プロジェクトのゴール

単なる「出席率を上げる」事業ではなく、**既存会員同士の相互理解 → JC活動の目的・意義の再確認 → 自分の役割の発見 → 仲間を支える意識の醸成**を通じて、挑戦する人を孤立させない加賀LOMの文化を作る議案を完成させる。

## 3. ロール定義（このプロジェクトで Claude に求める振る舞い）

- 役割：デザインストラテジスト ／ UXデザイナー
- 観点：ビジネス成果（参加率／継続率） × ブランド価値（加賀JCらしさ） × ユーザー体験（会員の体感） × 実装可能性（LOM規模で実現可能か）
- 思考順序：**Why → Who → What → How → Visual／Copy**
- アウトプット品質：そのまま資料に使えるレベル／構造的／根拠付き／プロフェッショナルかつ意思のあるトーン
- NG：一般論／ふわっとしたアイディアのみ／運用を考えない案／冗長な説明

## 6. 守ってほしい運用ルール

- **書式に手を入れない**：`reference/gian.htm` の CSS／mso-* 属性／テーブル構造は変更禁止。テキストセル内のみ編集する。
- **加賀JC2026理事長所信との整合を必ず確認**：`docs/01_project-context.md` の「上位方針との接続マップ」を参照。
- **既存会員向け事業として成立させる**：新人研修テンプレに引きずられない。`docs/02_proposal-background.md` の論点表を必ず参照。
- **アウトプットは必ず根拠付き**：事例・先例・データのいずれかを引いて主張する。出典URLは `04_case-references.md` に集約済み。

## 6-A. gian.htm フォーマット規則（固定・厳守）

**元本**: `Downloads/事業計画書2026/gian.htm`（加賀JC公式議案テンプレート Shift-JIS版）に準拠。以下のルールを一切破ってはならない。

### フォント
| 箇所 | フォント | サイズ | 備考 |
|---|---|---|---|
| 全セル本文・見出し | `"ＭＳ 明朝",serif` | **10.5pt** | 固定ピッチ。Pなし（ＭＳ Ｐ明朝 は使用禁止） |
| タイトルバー（最上段ピンク帯） | `"ＭＳ 明朝",serif` | **13.5pt** | white テキスト |
| mso-bidi-font-family | `"ＭＳ Ｐゴシック"` | — | 双方向テキスト用フォールバックのみ。本文には使用しない |

**禁止フォント**: `ＭＳ Ｐ明朝`, `ＭＳ Ｐゴシック`（bidi fallback 以外）, `ＭＳ ゴシック`, `游ゴシック`, `Hiragino`, `-apple-system` 等、Pの付くフォントおよびwebフォントをセル本文に使用してはならない。

### 文字種
- **英数字は半角**で記入する（全角英数字は使用しない）
- 英数字スパンには `lang="EN-US"` 属性を付与する

### 用紙・余白
- **用紙サイズ**: A4 (`595.3pt 841.9pt`)
- **余白**: 上下左右 `10.0mm`（`@page WordSection1` で定義済み）

### 色（セル内テキスト）
| 用途 | 指定値 |
|---|---|
| 通常テキスト | 指定なし（黒） |
| 訂正・修正箇所 | `color:red` |
| ハイパーリンク | `color:blue`（自動） |
| セクションヘッダーテキスト | `color:white`（FFAFDE背景上） |
| **禁止** | `#374151`, `#6b7280`, `#1e40af`, `#1f6feb`, `#0f172a`, `#1f2937`, `#92400e` など、テンプレート外の任意のhex色 |

### セル背景色
| 用途 | 指定値 |
|---|---|
| 通常セル | なし（白） |
| セクションヘッダー行 | `#FFAFDE`（ピンク） |
| タイトルバー背景 | `#7F7F7F`（グレー） |
| **禁止** | `#f3f4f6`, `#dbeafe`, `#fffbeb`, `#fef3c7`, `#d8345f`, `#0f172a` など任意の色 |

### 訂正・修正の表現方法
- 修正箇所は **赤字（`color:red`）で記入**する
- 下線は罫線と紛らわしいため **使用禁止**
- ステータスバッジ（CSS3 `border-radius` 等）は禁止。代わりに `（要更新）` のように赤テキストで示す

### 日付・時刻
- **日付**: 西暦表記 `YYYY年MM月DD日（曜日）` 形式。元号表記は使用しない
- **時刻**: **24時間法**で記入する（`午前`・`午後` の表記は使用禁止）
  - 例: 午後3時 → `15時`、午前10時 → `10時`

### テーブル構造
- `border:solid windowtext 1.0pt` / `mso-border-alt:solid windowtext .5pt` を維持する
- テーブル構造・colspan・rowspan・width属性は変更禁止
- `class="MsoNormalTable"` / `class="MsoTableGrid"` を維持する

### CSS3 / Webスタイル禁止
- `border-radius`, `box-shadow`, `gradient`, `flex`, `grid`, `transition` 等の CSS3 プロパティをセル内容に使用してはならない
- `#gian-dl-bar`（グローバルナビバー）は例外として web スタイルを使用してよい

## 6-B. コンプライアンス様式フォーマット規則（固定・厳守）

**対象ファイル**: `reference/参考資料/コンプライアンス受付表.htm` / `reference/参考資料/コンプライアンスチェックシート.htm`  
**元本**: `/Users/kawanishi-nori/Downloads/academy2/kisoku/コンプライアンス様式2025.xlsx`（加賀JC公式コンプライアンス様式 Excel版）に準拠。gian.htm と同レベルの厳密スタイル維持が必要。

### フォント
| 箇所 | フォント | サイズ |
|---|---|---|
| 全セル本文 | `'ＭＳ Ｐゴシック','MS PGothic',sans-serif` | **9pt**（本文）/ 10〜12pt（ヘッダ） |
| タイトル行 | 同上 | **12pt bold** |

### 構造維持規則
- HTML は Python（openpyxl）から自動生成。**手動で table 構造を書き替えてはならない**
- `colspan` / `rowspan` はExcelのマージセルから自動計算されたもの。変更禁止
- 行高さ（`height:19.35pt`）を維持する
- 列幅（`<colgroup>` の `<col style="width:X%">`）はExcel列幅を比率変換したもの。変更禁止

### 罫線
- `medium` 罫線 → `2px solid #000`
- `thin` 罫線 → `1px solid #777`
- 罫線なし → CSS border プロパティ自体を省略（`1px solid transparent` 等に置換しない）

### スタイル維持の対策
ファイルを修正する場合は必ず以下の手順：
1. Excel元本を `openpyxl` で読み込み（`data_only=True`）
2. `generate_sheet_html()` 関数でテーブルHTMLを再生成
3. プロジェクト固有データ（事業名・委員会名・担当者名）を置換
4. 既存HTML `<div class="form-body">` の内側のみ差し替え

### 禁止事項
- 手書きでテーブル行列を追加・削除すること
- `border-radius`, `box-shadow`, `gradient` 等のCSS3
- `-apple-system`, `Hiragino`, `Yu Gothic` 等のWebフォント
- `#f3f4f6`, `#0f172a` 等の任意のhex色（Excel fillからRGB変換したもの以外）
